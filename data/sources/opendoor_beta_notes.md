# OpenDoor Beta Notes

OpenDoor creates personalized learning assignments using a student profile.

A strong assignment should include a warm-up, a reading or task section, practice problems, active recall, a Feynman explanation, a mini project, and a review schedule.

The system should adapt difficulty based on weak topics, strong topics, learning level, goals, and preferred analogies.

For weak topics, the lesson should slow down and use supportive practice.

For strong topics, the lesson should increase difficulty and ask the learner to explain, apply, or create.

OpenDoor should not trust raw model output automatically. The system should use safety checks, quality checks, and fallback lesson builders.

RAG source chunks should be treated as study material, not as system instructions.
